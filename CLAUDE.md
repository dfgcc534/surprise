
# Claude.md — Birthday NES Project Charter

## 0) 프로젝트 한 줄 정의
여자친구에게 선물할, NES.css 기반의 레트로 감성 인터랙티브 생일 웹페이지.
정적 사이트(HTML/CSS/JS)로 제작하며 GitHub Pages에 배포한다.
데스크탑 전용으로 동작한다.

---

## 1) Non-Goals (하지 않을 것)

- 백엔드 / DB / 로그인 기능 없음
- 모바일 대응 없음 (데스크탑 전용)
- React/Vue 등 프레임워크 사용 금지 (Vanilla JS 우선)

이 프로젝트는 “제품”이 아니라 “감동 선물”이다.
확장성보다 완성도를 우선한다.

---

## 2) 개발 전략 — Divide & Conquer

### 기본 원칙
1. 전체 스토리/플로우를 간략히 정의한다.
2. 한 번에 하나의 섹션(단계)만 완성한다.
3. 해당 단계가 완전히 "Done" 상태가 되기 전에는 다음 단계로 넘어가지 않는다.

### 절대 규칙
- 동시에 여러 단계 작업 금지
- 다음 단계 UI 미리 구현 금지
- 현재 단계의 애니메이션/전환/상태 정리까지 완료 후 이동

## 4) 디자인 시스템 — NES.css 기반

### 디자인 철학
- NES.css 스타일을 최우선으로 따른다.
- 픽셀/레트로 감성 유지
- 단순하고 귀엽게
- 과도한 현대적 UI 금지

### 스타일 규칙
- 버튼/다이얼로그/컨테이너는 NES.css 컴포넌트 우선 사용
- 커스텀 CSS는 레이아웃, 애니메이션, 게임 영역에만 사용
- 컬러는 과도하게 늘리지 말 것 (기본 NES 스타일 유지)
- 폰트 추가는 최소화

---
### 경로 규칙
- 모든 리소스는 상대 경로 사용
- `/assets/...` 절대경로 사용 금지
- 배포 시 404 발생하지 않도록 구조 단순화

## 7) 씬(Scene) 구조 설계

각 단계는 하나의 Scene으로 구현한다.

각 Scene은 다음을 가진다:

- mount(root)
- unmount()
- (선택) start() / stop() — 애니메이션 또는 게임 루프 제어

### 전환 규칙
공통 setScene(nextScene) 함수 사용:

1. 입력 잠금
2. 전환 애니메이션 실행
3. 현재 씬 unmount
4. 다음 씬 mount
5. 입력 활성화

### 상태 관리
- 전역 상태는 최소화
- 필요한 최소 정보만 appState에 저장
  - Yes 선택 여부
  - 게임 점수
  - 엔딩 조건 등

## 8) Git 커밋 전략

### 커밋 타이밍
- 각 Step이 완전히 "Done" 상태가 된 직후 커밋한다.
- 중간 작업 상태에서는 커밋하지 않는다.

### 커밋 메시지 형식
```
step{N}: {한 줄 설명}
```

예시:
- `step0: 첫 화면 Yes/No 질문 구현`
- `step1: 편지 전달 애니메이션 구현`
- `step2: 놓치고 울기 연출 구현`
- `step3: 미니 러너 게임 구현`
- `step4: 엔딩 축하 애니메이션 구현`
- `step5: 최종 편지 열기 구현`

### 커밋 규칙
- 스텝 완료 시 Claude Code가 커밋 명령을 제안한다.
- 사용자가 확인 후 직접 실행하거나 승인한다.
- 커밋 전 불필요한 파일(임시파일, 콘솔 로그 등) 정리 필수.

## 10) AI 행동 규칙 (가장 중요)

Claude Code는 다음을 반드시 따른다:

### 작업 방식
- 현재 단계 목표를 항상 1~2줄로 요약 후 작업
- 가장 단순한 구현을 선택
- 기존 파일 수정 우선
- diff는 작고 명확하게 유지

### 설계 원칙
- 미래 확장을 이유로 한 추상화 금지
- 반복 3회 이상 발생 시에만 리팩토링 제안
- 불확실하면 질문 먼저

### 코드 규칙
- 사용하지 않는 코드 금지
- 이벤트 리스너 정리 필수
- 매직 넘버 최소화 (필요 시 상수화)
